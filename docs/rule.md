# Phaser Supermarket Quest ゲームルール・仕様

## 1. このファイルの役割

このファイルには、実際にプレイするときのルール、操作、勝敗条件、UI、会話進行、収集仕様、会計判定など、**ゲームとしてどう動くか**をまとめる。

ここで扱う内容は、主に次のとおりである。

- プレイヤーが何をすればよいか
- どうすればクリアになるか
- 何をすると失敗になるか
- 操作方法とインタラクション方法
- NPC、会話、ヒント、アイテム収集の仕様
- 現在の実装に基づく判定と制約

ゲームの目的や教育的背景、なぜこの構造にしているのかといった企画意図は `docs/plan.md` に分けて記述する。

## 2. ゲームの基本概要

このゲームは、スーパーの中を移動しながら必要な食材を集め、最終的にレジで正しく会計することを目指す探索型タスクゲームである。

基本構造は次のとおりである。

- 毎回ランダムで 1 つの料理が選ばれる
- 制限時間は 5 分である
- プレイヤーは店内を歩き回り、店員に話しかけたり食材を拾ったりする
- 最終的に、選ばれた料理に必要な食材を過不足なく集めてレジに行く

単に必要な物を全部拾えばよいわけではなく、**不要な物を余分に持ったまま会計すると失敗になる**点が重要である。

## 3. ゲーム開始時の状態

ゲーム開始時には、次の状態でプレイが始まる。

- 制限時間は `5:00` から開始する
- 料理は候補の中からランダムで 1 つ選ばれる
- プレイヤーの所持品は空である
- 食材はすべて店内に配置されている
- 必要食材の一覧は内部的には決まっているが、最初から全部見えているわけではない
- ヒントログは空である
- 画面上のタスク表示は「料理に必要な食材を集めよう」系の内容になる

## 4. クリア条件と失敗条件

### 4-1. クリア条件

次の条件を満たした状態でレジに行くとクリアになる。

- その回で選ばれた料理に必要な食材をすべて持っている
- 必要でない食材を 1 つも持っていない
- 制限時間が残っている

つまり、**必要食材を全部集めること**と、**不要な物を持ち込まないこと**の両方が必要である。

### 4-2. 失敗条件

次のどちらかに当てはまると失敗になる。

- 制限時間が 0 になる
- レジで会計した時点で、必要食材が不足している、または不要食材を余分に持っている

### 4-3. 失敗にならないもの

次の行動は、その場で即失敗にはならない。

- NPC との会話で間違った選択肢を選ぶ
- 不要な食材を拾う
- 一度拾った食材をバッグから戻す

ただし、不要な食材を持ったまま会計に進むと失敗になるため、探索中の判断は最終判定に影響する。

## 5. 操作方法

現在の操作方法は次のとおりである。

- `←` `→` `↑` `↓` でプレイヤーを移動する
- `SPACE` で近くの対象に応じたアクションを行う
- 会話中の選択肢は画面上のボタンをクリックして選ぶ
- バッグ内のアイテムはクリックすると棚に戻せる
- リザルト画面ではクリックで再スタートできる

## 6. インタラクションの基本ルール

プレイヤーは、近くにある対象に応じて異なる行動を取る。

- NPC の近くでは `SPACE: 話す`
- 食材の近くでは `SPACE: ひろう`
- レジの近くでは `SPACE: レジ`

この表示は状況に応じて自動で切り替わる。会話中は通常の移動・探索行動は止まり、会話 UI が前面に出る。

## 7. プレイの基本的な流れ

1 回のプレイは、おおむね次の流れで進む。

1. ランダムに料理が選ばれる
2. プレイヤーはスーパー内を移動する
3. 店員に話しかけて、料理や食材に関するヒントを得る
4. 店内を探索して食材を拾う
5. 必要食材がそろったらレジへ向かう
6. 会計結果に応じてクリアまたは失敗になる
7. 結果画面で必要食材の英単語を復習し、再スタートする

必要食材がそろっても自動ではクリアにならず、**必ずレジに行って会計する必要がある**。

## 8. マップと空間仕様

### 8-1. ゲーム世界のサイズ

- ゲーム世界全体は `1920 x 1080`
- 表示用のゲーム画面は `960 x 540`
- カメラはプレイヤーを追従する

つまり、プレイヤーは固定画面 1 枚の中だけではなく、店内全体を移動して探索する構造になっている。

### 8-2. 売り場構成

マップ上には、少なくとも次の売り場ラベルが存在する。

- 野菜
- 缶詰
- 乳製品
- ハーブ
- スパイス

### 8-3. 障害物

- 棚は衝突判定を持つ
- プレイヤーは棚をすり抜けられない
- 移動時には通路を回って目的地へ向かう必要がある

### 8-4. レジ

- マップ内にはレジが 1 か所ある
- その近くで `SPACE` を押すと会計判定が始まる

## 12. 会話システム

### 12-1. 会話の開始

- NPC に近づいて `SPACE` を押すと会話が始まる
- 会話中はプレイヤーの移動が止まる
- 会話 UI が画面前面に表示される

### 12-2. 会話 UI

会話画面では、主に次の要素が表示される。

- 直近の会話履歴
- 現在の問いかけ
- 選択肢ボタン

選択肢ボタンは最大 5 個まで表示される。

### 12-3. 会話ターン数

- ヒント NPC の会話は 2〜4 ターンでランダム
- 雑談 NPC の会話は 2〜3 ターンでランダム

### 12-4. 選択肢数

- 各ターンの選択肢数は 2〜5 個でランダム
- 正答 1 つと、会話文脈に合わないダミー選択肢が混ざる

### 12-5. 正解・不正解時の挙動

- 正しい選択肢を選ぶと次のターンへ進む
- 最終ターンで正解した場合は会話が成功終了する
- 間違った選択肢を選ぶと、その会話はその場で終了する

会話を間違えてもゲーム自体は終わらない。会話が閉じるだけであり、再び移動して別の行動を取れる。

### 12-6. タイマーとの関係

会話中はタイマーが進まない。したがって、会話を開いている間は制限時間が停止する。

## 13. ヒント解放の仕様

### 13-1. ヒント対象の選ばれ方

- ヒント NPC は、まだ回収していない必要食材の中から 1 つを対象にする
- すでに対象として注目している必要食材が残っていれば、その対象を維持する場合がある
- もう未回収の必要食材がなければ、新しいヒント会話は出ない

### 13-2. ヒントの種類

ヒントは次の 4 系統に分かれる。

- 食材名そのもの
- 売り場
- 色
- 種類

### 13-3. 解放のタイミング

- ヒント会話の途中で正解しても、その時点ではまだ報酬は確定しない
- **最終ターンで正解したときだけ** 対象食材が解放される
- 解放されると、その食材の表示名が分かる状態になる
- 同時にヒントログにも記録される

## 14. 食材の見え方と収集仕様

### 14-1. 見え方

- 未解放の食材はラベルが `?` で表示される
- 解放済み、または一度収集した食材は英単語が見える状態になる

### 14-2. 収集

- 食材に近づいて `SPACE` を押すと拾える
- 拾った食材はマップ上から消え、バッグに入る
- バッグには現在所持している食材が表示される

### 14-3. 現在の実装上の重要点

README の説明からは、解放済みの必要食材だけを拾えるようにも読める。しかし、**現行実装では未解放の食材でも近づけば拾える**。

つまり現在は、ヒントは回収の必須条件というより、探索を助けるための支援機能として働いている。したがって、会話を使わずに食材を総当たりで拾い、不要分だけ戻してから会計する戦略も理論上は成立する。

### 14-4. 戻す操作

- バッグ内のアイテムはクリックで元の棚側に戻せる
- 戻した食材は所持品から削除され、再びマップ上に出現する

この仕様により、不要な物を拾ってしまっても、会計前に整理し直せる。

## 15. バッグと HUD の仕様

### 15-1. 外側ページの表示

`index.html` 側では、少なくとも次の表示がある。

- ゲームタイトル
- ゲームの短い説明文
- 時間表示カード
- タスク表示カード

### 15-2. タスク表示

タスク表示は状態に応じて切り替わる。

- 必要食材が残っている間は、料理の食材を集める指示が出る
- 必要食材をすべて持ったら、レジに向かう指示に切り替わる

### 15-3. バッグ表示

ゲーム画面内にはバッグ HUD があり、次の要素を持つ。

- バッグタイトル
- `クリックで戻す` という補助テキスト
- 所持数バッジ
- 食材スロット一覧

所持数は `現在の所持数 / 全食材数` の形式で表示されるため、最大値は 12 である。

### 15-4. 会話 UI

ゲーム画面内には、会話用のオーバーレイ UI がある。

- 会話背景パネル
- 会話履歴テキスト
- 現在の問いテキスト
- 選択肢ボタン

### 15-5. システムフィードバック

ゲーム中には、状況に応じて次のようなフィードバックが出る。

- 会話成功時のメッセージ
- 会話失敗時のメッセージ
- 食材入手メッセージ
- 食材を戻したメッセージ
- 必要食材がそろったことを知らせるメッセージ

## 16. 会計判定の詳細

会計時には、所持品が次の 2 つに分けてチェックされる。

- 必要なのに持っていない食材
- 持っているが必要ではない食材

クリアになるのは、この両方が空のときだけである。

したがって、次のケースはすべて失敗になる。

- 必要食材が 1 つでも足りない
- 必要食材は全部あるが、不要食材を余分に持っている
- 必要食材が足りず、なおかつ不要食材も持っている

## 17. リザルト画面

ゲーム終了後には結果画面が表示される。

- クリア時は成功メッセージが表示される
- 時間切れ時は不足食材が表示される
- 会計失敗時は不足食材と余分な食材が表示される
- 必要食材の英単語が復習用に表示される
- クリックでシーンが再起動し、最初からやり直せる

## 18. 実装上の補足と注意点

現在の実装をルールとして読むとき、次の点に注意が必要である。

- 会話選択はキーボード番号ではなくクリック操作で行う
- 会話中はタイマーが止まる
- 不要食材も拾えるが、会計に持ち込むと失敗要因になる
- バッグから戻す操作があるため、探索中の誤取得は後から修正できる
- ヒント解放は最終正答時のみであり、中間正答だけでは報酬にならない
- README は簡略説明になっており、現行実装では「必要食材を集める」だけでなく、レジ会計と不要食材ゼロもクリア条件に含まれる

## 19. ルールを一文でまとめると

**5 分以内に、店員との英語会話で手がかりを集めながら、ランダムに選ばれた料理に必要な 4 つの食材だけを正しく集め、余計な物を持たずにレジで会計できればクリアとなるゲーム**である。
