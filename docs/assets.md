# Phaser Supermarket Quest アセット準備メモ

## 1. このファイルの役割

このファイルには、このゲームで使用する視覚・音・UI 素材について、**何を用意すべきか**、**現状どこまで実装済みか**、**どの素材を優先して準備すると効果が大きいか**をまとめる。

企画意図は `docs/plan.md`、ゲームルールと仕様は `docs/rule.md`、事業背景は `docs/bizplan.md` に分け、このファイルでは主に**制作素材の棚卸し**を扱う。

## 2. 現在の実装で実際に使っているアセット

現時点で、コード上で外部ファイルとして読み込んでいる素材は多くない。

- プレイヤー用スプライトシート
  - `assets/player/shopper-sheet.png`
  - `src/game.js` の `preload()` で読み込まれている

一方で、次の要素は現在は画像ファイルではなく、コード上で簡易生成されている。

- 棚
- NPC 見た目
- 食材オブジェクト
- 背景の大部分
- レジの見た目
- 会話 UI パネル
- バッグ UI

つまり、**今のゲームは最小限の外部アセットで動いているが、見た目の多くは仮実装に近い**状態である。

## 3. このゲームで準備を検討したいアセットの全体像

このゲームで準備対象になりやすいアセットは、次の 6 系統に分けられる。

- キャラクター系
- マップ・背景系
- 食材・オブジェクト系
- UI 系
- 演出系
- 音系

## 4. 優先度が高いアセット

### 4-1. プレイヤーアニメーション素材

現在はプレイヤー用スプライトシートがあるため、最低限の移動は成立している。
ただし、長く使う前提なら次の整理があるとよい。

- 歩行 4 方向が分かるスプライトシート
- 待機フレームが明確な構成
- 将来的に会話中やクリア時に使える表情差分

必要理由:

- プレイヤーは常に画面中央付近で目に入りやすい
- 動きの品質がゲーム全体の印象に直結する

### 4-2. 食材アイコンまたはスプライト

現在の食材は、緑の丸とテキストで表現されている。
学習ゲームとしては、**単語と見た目を結びつける素材**があると効果が大きい。

現在の食材は 12 個ある。

- eggplant
- tomato
- onion
- chickpeas
- coriander
- yogurt
- garlic
- lentils
- cheese
- paprika
- turmeric
- cumin

必要理由:

- 単語、色、種類、見た目の結びつきが強くなる
- `?` から正体が分かった時の達成感を出しやすい
- バッグ UI や結果画面にも流用しやすい

推奨形式:

- 各食材の個別 PNG
- もしくは食材アイコンをまとめたスプライトシート

### 4-3. 売り場・棚・レジのマップ素材

現在の店内は、矩形や簡易テクスチャで構成されている。
探索ゲームとしての説得力を上げるには、次の素材の優先度が高い。

- 床タイル
- 棚スプライト
- レジスプライト
- 売り場サイン
- 通路の視認性を上げる床模様

必要理由:

- どこが何売り場かを視覚で理解しやすくなる
- 探索時の空間記憶を助ける
- スーパーらしさが一気に出る

### 4-4. NPC スプライトまたは立ち絵

現在の NPC は色違いの簡易円形キャラで表現されている。
NPC が 10 人いるため、識別性の向上は効果が大きい。

現在の NPC:

- Mia
- Ken
- Sora
- Lena
- Noah
- Emma
- Riku
- Ava
- Yuna
- Taro

必要理由:

- 誰がヒント役か、誰が雑談役かを印象で覚えやすくなる
- 会話ゲームとしての人物感が出る
- 名前だけでなく見た目でも記憶できる

推奨方針:

- 最低限は 10 人分の顔差分または立ち姿
- コストを抑えるなら 2 系統
  - ヒント NPC 用デザイン
  - 雑談 NPC 用デザイン

## 5. あると効果が大きいアセット

### 5-1. UI パネル素材

現在の UI は矩形とテキスト中心で成立しているが、次の素材があると見栄えが安定する。

- 会話ウィンドウ背景
- 選択肢ボタン背景
- バッグ枠
- タスク表示パネル
- ヒントログパネル
- 結果画面パネル

必要理由:

- 情報のまとまりが視覚的に分かりやすくなる
- 教材っぽさではなく、ゲームらしい完成感が出る

### 5-2. 料理カード・目標表示素材

ゲーム開始時や HUD に、料理目標を視覚化する素材があると理解が速い。

- 料理カード枠
- 料理のシルエットまたは写真風イラスト
- 料理ごとの小アイコン

現在の料理候補は 3 つ。

- Khorkhog
- Moqueca
- Harira

必要理由:

- その回の目標が一目で分かる
- 未知の料理という設定を印象づけやすい
- クリア後の復習演出にも使える

### 5-3. アイコン類

細かい補助アイコンがあると UI が整理しやすい。

- タイマーアイコン
- バッグアイコン
- 会話アイコン
- レジアイコン
- 正解 / 不正解アイコン
- ヒント種別アイコン
  - 食材名
  - 場所
  - 色
  - 種類

## 6. 後から追加してもよいアセット

### 6-1. 効果音

現在の実装では、音がなくてもゲームは成立する。
ただし、体験品質を上げるなら次が有効。

- アイテム取得音
- 会話開始音
- 正解音
- 不正解音
- 会計成功音
- 会計失敗音
- ボタン hover / click 音

### 6-2. BGM

- タイトル BGM
- プレイ中 BGM
- クリア時ジングル
- 失敗時ジングル

学習ゲームなので、主張しすぎない落ち着いたループ音源が向いている。

### 6-3. 演出素材

- きらめきエフェクト
- 正解時の小さなパーティクル
- 会話成功時の吹き出し演出
- クリア画面の装飾

## 7. 今のゲームに対して最低限そろえるとよいセット

コストを抑えつつ、見た目と学習体験を大きく改善したいなら、まず次の順で用意するとよい。

- プレイヤー用スプライトの品質調整
- 12 食材の個別アイコン
- 棚、床、レジの基本マップ素材
- 10 NPC の識別しやすい見た目
- 会話 UI とバッグ UI のパネル素材
- 取得音、正解音、会計成功音の 3 種類の効果音

## 8. 実装優先度の提案

### 優先度 A

ゲーム性と学習体験に直結する。

- 食材画像
- 棚 / 床 / レジ素材
- NPC 見た目差分

### 優先度 B

体験の完成度を大きく上げる。

- UI パネル
- 料理カード
- 基本効果音

### 優先度 C

後から追加しやすい。

- BGM
- 演出エフェクト
- 細かい装飾アイコン

## 9. 推奨フォルダ構成

将来的に素材が増えるなら、次のように分けると管理しやすい。

- `assets/player/`
- `assets/npc/`
- `assets/items/`
- `assets/map/`
- `assets/ui/`
- `assets/audio/bgm/`
- `assets/audio/sfx/`

## 10. 制作時の注意点

- 1 つの食材画像を、マップ上・バッグ内・結果画面で再利用できる設計にする
- NPC は役割の違いが見た目にも出るようにする
- 売り場素材は英単語学習を邪魔しない程度に視認性を確保する
- UI 素材は派手さより可読性を優先する
- 学習ゲームなので、背景や BGM が情報理解を邪魔しないバランスにする

## 11. 現時点での要約

現状のこのゲームは、**プレイヤーのスプライト以外は多くが簡易表現で成立している**。
そのため、今後アセットを用意するなら、優先順位は次の順がよい。

- 食材の見た目
- マップの見た目
- NPC の見た目
- UI 素材
- 音

特に、**食材画像とスーパーらしい売り場素材**は、このゲームの学習体験と世界観の両方に強く効く。

## 12. 現時点の採用候補

現時点では、店内の背景・内装素材の有力候補として次を採用候補にする。

- `LimeZu / Modern Interiors`
- URL: `https://limezu.itch.io/moderninteriors`

この候補を選ぶ理由は次のとおりである。

- モダンな indoor / store 系の表現に強い
- grocery store を含む現代的な店舗内装へ流用しやすい
- 家具、床、壁、装飾のバリエーションが多い
- 今後スーパー以外の現代施設に拡張する場合にも使い回しやすい

一方で、使用時には次を確認する。

- 無料版と完全版の違い
- 利用ライセンスとクレジット表記条件
- 実際に使うタイルサイズと既存プレイヤー素材との相性

当面は、この素材を前提に `assets/map/` や NPC、UI 素材との見た目の整合を考えていく。
