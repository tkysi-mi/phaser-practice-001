# Phaser Supermarket Quest 企画意図・学習設計

## 1. このファイルの役割

このファイルには、このゲームが何を目指しているのか、なぜそのような構造になっているのか、英語学習としてどのような体験を作りたいのかをまとめる。

ここで扱うのは主に次の内容である。

- ゲーム全体の目的
- 学習者にどのような経験をしてほしいか
- なぜスーパーという舞台を選んでいるのか
- なぜ料理、食材、売り場、店員との会話を組み合わせているのか
- CEFR A2 相当の can-do 的な考え方とどうつながるか

一方で、実際の操作方法、勝敗条件、UI の細かい仕様、会話ターン数、判定ロジックなどは `docs/rule.md` にまとめる。

## 2. このゲームが目指しているもの

このゲームが目指しているのは、英単語を単体で暗記させることではなく、**目的のある行動の中で英語を使わせること**である。

プレイヤーは、ただ問題を解くのではなく、次のような状況に置かれる。

- 何の料理を作るのかを把握する必要がある
- その料理に何が必要かを人に聞くと有利になる
- 得た情報をもとにスーパーの中を探す必要がある
- 集めたものが正しいかを最後に確認される

つまりこのゲームでは、英語はテストのための記号ではなく、**ゲームを前に進めるための道具**として登場する。これが企画の中心にある考え方である。

## 3. なぜこのゲームを英語学習ゲームとして作るのか

一般的な語彙学習は、英単語と日本語訳を 1 対 1 で対応させるだけになりやすい。しかしその方法だけでは、実際の場面で何を聞けばよいか、どの情報が重要か、何を根拠に選ぶかが育ちにくい。

このゲームでは、学習者に次の流れを体験させたい。

- まず、分からない状態に置かれる
- 次に、誰かに聞いた方が先に進みやすい状況になる
- その会話の中で、意味のある質問表現に触れる
- 手がかりを頼りに空間の中で探索する
- 結果として、単語、場所、特徴、用途を関連づけて覚える

この流れによって、単なる丸暗記ではなく、**理解して使うための語彙経験**を作ることを狙っている。

## 4. なぜスーパーという舞台なのか

スーパーという舞台には、学習ゲームとして扱いやすい理由がいくつかある。

- 売り場という分かりやすい分類がある
- 食材は色、種類、場所、用途など複数の手がかりで表現できる
- `探す`、`聞く`、`選ぶ`、`集める`という行動が自然につながる
- 買い物という日常的なテーマなので、A2 レベルの学習者にも文脈が伝わりやすい

さらに、スーパーは目の前に物があるのに、正しい物を選ばなければならない状況を作りやすい。これは、英語を読めたかどうかだけでなく、**読んだ内容を判断に使えたか**まで体験させるのに向いている。

## 5. なぜ未知の料理を題材にするのか

このゲームでは、最初から材料が完全に分かっている料理ではなく、学習者にとって未知である可能性がある料理名を起点にしている。

これには明確な意図がある。

- 最初から答えを知っている前提を崩せる
- `知らないから聞く` という会話の動機が自然に発生する
- 推測、確認、再確認という流れを作りやすい
- 異文化的な要素が入り、単語学習を文脈化しやすい

もしプレイヤーが最初から料理の材料を全部知っているなら、会話は不要になりやすい。未知の料理を使うことで、英語による情報収集がゲームの中心行動になる。

## 6. このゲームで作りたいプレイヤー体験

企画上は、プレイヤーに次のような感覚を持ってほしい。

### 6-1. 分からないけれど、聞けば前に進める感覚

完全に知識がない状態でも、会話の中で手がかりを得て少しずつ前進できる感覚を重視する。学習者に、英語が完璧でなくても必要なことは聞ける、という感覚を持たせたい。

### 6-2. 英語を読んで行動が変わる感覚

どの選択肢を選ぶかによって会話結果が変わり、その結果が探索ルートや収集行動に影響する構造にすることで、英語理解がそのままゲーム上の意思決定になる体験を作る。

### 6-3. 単語を点ではなくネットワークで覚える感覚

たとえば 1 つの食材について、次の情報が結びついた状態で記憶に残るようにしたい。

- 名前
- 売り場
- 色
- 種類
- 料理との関係

これは語彙を単独で覚えるより、後の想起を助けやすい。

### 6-4. 失敗しても学習が続く感覚

即時失敗や強い罰を前面に出すのではなく、試して、間違えて、また次に挑戦できる構造にしたい。学習者にとって重要なのは、完璧に答えることよりも、繰り返し触れながら理解を深めることである。

## 7. 学習設計としての中核

このゲームの学習設計は、大きく分けると次の 4 つの柱で考えられる。

### 7-1. タスク先行

先に `料理を完成させる` というタスクがあり、そのタスクの達成のために英語を使う。つまり、言語学習が主題でありながら、プレイヤーの意識は常に `何をすればクリアできるか` に向く。

この構造によって、英語表現は単なる問題文ではなく、目的達成のための選択肢として機能する。

### 7-2. 情報ギャップ

プレイヤーは最初から必要情報を持っていない。だからこそ、質問する意味が生まれる。情報ギャップがあることで、会話が形式的なものではなく、必要に迫られた行為になる。

### 7-3. 文脈付き反復

同じ語彙や関連表現を、次のような複数の場所で反復させる。

- 会話の選択肢
- NPC の返答
- マップ上の探索
- バッグ表示
- 結果画面の復習

こうすることで、認知負荷を上げすぎずに再接触を増やす。

### 7-4. 受容から行動への接続

英語を理解しただけでは終わらず、その理解をもとに `どこへ行くか`、`何を拾うか`、`誰に聞くか` を決める必要がある。この接続があることで、読解・理解と行動判断が結びつく。

## 8. CEFR A2 / can-do 的な位置づけ

README ではこの作品を CEFR A2 向けとしている。現在の実装と企画意図を踏まえると、このゲームは特に次のような can-do 的経験を支えるものとして整理できる。

- 身近な必要に関する短い表現を読んで理解できる
- 非常に短いやり取りの中で、自然な応答を選べる
- 色、場所、種類などの具体的な情報を手がかりにできる
- 買い物や案内に近い文脈で、必要な情報を得ようとできる
- 単純な質問表現を目的に応じて選択できる

ここで重要なのは、このゲームが自由英作文や長い発話を要求するものではないという点である。現在の重心は、**自由産出よりも、理解・選択・状況判断**にある。

したがって、このゲームは次のような段階づけの中で位置づけると分かりやすい。

- 最初に、短い英語を読んで意味を取る
- 次に、状況に合う表現を選ぶ
- その後、語彙やヒントを行動に変換する
- 最終的に、必要な物をそろえて目的を達成する

## 9. このゲームで伸ばしたい力

企画上、特に育てたいのは次のような力である。

### 9-1. 語彙理解

食材名、売り場名、色、種類など、具体物に結びついた語彙を理解する力。

### 9-2. 機能表現の理解

`What ingredients does it usually have?`、`Where can I find it?` のように、情報を聞き出すための実用的な表現を理解する力。

### 9-3. 状況判断

どの表現が今の会話の流れに合うかを見極める力。これは単なる単語知識ではなく、会話の自然さを判断する力でもある。

### 9-4. 情報統合

名前、場所、色、種類といった断片情報をつなげて、何を探すべきかを判断する力。

### 9-5. 反復を通じた定着

同じ単語に複数回、異なる形で触れることで、意味と使いどころを少しずつ定着させる力。

## 10. 逆に、このゲームが主目的にしていないもの

企画を明確にするために、現時点で主目的にしていないものも書いておく。

- 長文読解を中心にした学習
- 文法説明を前面に出した学習
- 自由記述や自由発話の厳密評価
- ネイティブ運用レベルの会話生成
- スコア最適化を主目的とする競技的ゲーム性

もちろん将来的に拡張余地はあるが、現時点では `短い英語を理解し、適切に選び、行動につなげる` ことが主軸である。

## 11. 現在の実装との関係

現在の実装は、企画意図のすべてを完全に表現しているわけではないが、少なくとも次の点では意図と一致している。

- 未知の料理を起点に情報収集する構造
- ヒント NPC と雑談 NPC の役割差
- 売り場、色、種類、食材名を学習手がかりにする設計
- 会話理解が探索行動に影響する構造
- クリア後に必要語彙を復習できる流れ

一方で、将来的に整理の余地がある点もある。

- README の説明と実装の厳密な一致
- ヒント解放と食材回収の関係の明文化
- can-do の記述と個別会話パターンの対応づけ
- 語彙学習中心なのか、会話方略まで含めるのかの明示

- 現在は未解放食材でも拾えるため、会話が進行の必須条件にはなっていない点の整理
- 不要食材を戻せるため、総当たり回収から整理して会計する戦略も成立する点の整理

## 12. この企画を一言で言うと

**スーパーでの買い物という身近なタスクを通して、英語を読んで、聞いて、選んで、行動に変える練習をさせる、小さなタスク型英語学習ゲーム**である。
